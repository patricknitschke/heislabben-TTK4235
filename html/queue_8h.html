enum order_t

Queue order tags for variables in queue array.

These tags correspond to the order buttons outside the elevator. In the case of an order from the inside of the elevator, the order is treated as a order_t in both directions.

Sets the elevator direction according to target_floor.

If target_floor is greater than the elevator floor, direction is set upwards and vice versa. If the target_floor is the same as the current floor, the elevator stops.

◆ queue_stop_n_serve_order()

int queue_stop_n_serve_order ( void  )

Identifies when to serve a customer and serves a customer if certain conditions are met.

If any of the below conditons are satisfied, queue_clear_orders_at_floor is called to remove orders and turn off lights at the elevator's current floor.

Condtions to which a customer can be served:

  • Direction is up and same direction order exists
  • Direction is up and no orders above
  • Direction is down and same direction order exists
  • Direction is down and no orders below
  • Last order in queue.
Returns
1 if the conditions to serve a customer are met. 0 otherwise.
Note
If the elevator is not on a valid floor, the function will always return 0.
Warning
Despite its name, this function does not physically stop the elevator. However, removing orders from queue can result in a stop after queue_chase_target is called.
